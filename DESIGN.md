# 《叠个果王》设计文档(存档)

> 一句话玩法:从叠放的水果卡片中挑出相同水果,三张相同消除并生成实体水果,再通过物理碰撞不断合成,最终合出"果王"。
> 宣传语:点三个,掉一个,碰两个,合成果王!

融合:《羊了个羊》叠层选择+卡槽压力 / 《合成大西瓜》物理碰撞 / 《2048》升级成长 / 《挑一挑》寻找解锁感。

## 三个区域

1. **叠层区**(上):互相遮挡的水果卡片,只有完全未被上层牌压住的可点。当前共有 19 级水果链；每关从蓝莓起步并逐步加入更高阶卡片。
2. **卡槽区**(中):7 格。相同卡片自动归组;凑满 3 张→爆裂消除→生成实体水果落入合成箱。7 张无法消除即失败。
3. **合成箱**(下):透明物理箱。左右滑动选择掉落位置,两个相同水果碰撞→合成高一级。

## 合成路线(19 级)

蓝莓→葡萄→樱桃→草莓→柠檬→猕猴桃→牛油果→番茄→橙子→苹果→梨→桃子→芒果→香蕉→椰子→菠萝→哈密瓜→西瓜→黄金果王

## 胜负

- 胜利:合成关卡目标水果(每日挑战:清空全部卡片+合成黄金果王)。
- 失败:①卡槽 7 张满且无法三消;②合成箱水果超警戒线持续 2 秒(超线先进入红色警报,给自救机会)。

## 分数

- 三消:水果等级 × 30
- 合成:50 × 2^(等级-1)(樱桃100 … 黄金果王12800)
- 连击(2 秒内连续三消/合成):2连×1.25 / 3连×1.5 / 4连×2 / 5连×2.5 / 6连+×3
- 连锁演出:2连蓝色冲击波 / 3连紫色拖尾 / 4连金色慢动作 / 5连+全屏彩虹;文案:太甜了!/果汁爆发!/超级连锁!/果王降临!

## 道具(首版 4 个)

撤回(每局免费1次)/ 洗牌(不改遮挡层级)/ 榨汁机(水果降一级)/ 万能果(万能三消)。

## 特殊卡片(后续)

冰冻卡、锁链卡、问号卡、传送带、石头水果(只能被高级合成冲击波破坏)、果汁炸弹。

## 关卡难度参数

卡片种类 3→6 / 叠层深度 2→6 / 初始可点数量 / 合成箱宽度 / 目标水果等级 / 特殊卡数量。
生成规则:每种普通卡数量为 3 的倍数;至少一条通关路径(自动求解器验证);前五次点击至少可组一组三消;至少两种可行解法。

## 章节

新手 4 关(点击→合成→卡槽压力→警戒线)→ 清晨果摊(1-20)→ 夏日果园(21-50)→ 水果夜市(51-100)。

## 商业化 / 留存(摘要)

激励广告仅玩家主动触发(复活/清顶/撤回/翻倍),每局最多复活一次;内购只卖外观与去广告。图鉴多外观、签到、赛季换皮(甜品季/海洋季/星球季…)。每日挑战固定种子+操作回放防作弊,战绩卡分享("我只差一个草莓就能合出果王,你能过吗?")。

## 三大爽点

1. 卡槽将满时最后一张卡触发三消,死里逃生。
2. 一次掉落触发连锁合成,连环爆炸(短视频传播点)。
3. 最后一张卡→三消→物理连锁→黄金果王诞生的完整高潮。

## 首版 MVP

1 主题 / 9 级水果 / 60 主线关 / 每日挑战 / 4 特殊卡 / 4 道具 / 排行榜 / 图鉴 / 复活 / 分享。
技术:卡片记录 id/fruitType/position/layer/coveredBy/state;物理引擎(Matter.js 等);合成时先锁定两果再生成防重复;每日挑战固定随机种子。

### 玩法修正（2026-07-16，第二次）

重构时丢失/写错的旧版核心机制已找回：

- 磁吸修复：原公式越近吸力越小且不足以克服 Matter 地面静摩擦（约 108 px/s²），残局同级水果碰不到一起；改回旧版"越近越强"曲线（150~430 px/s²，速度增量实现）；
- 脱困修正：同级水果被夹住拉不动 1.1s 后，朝对方横向滚动并增加角速度，避免"三明治"软锁；
- 残局判负找回"两两合并推演"（`canReachTarget`），并排除顶级水果不可合并的误判；
- 玩法计时全部改为游戏时间 `elapsed`（水果年龄/警戒线/残局/连击窗口）：此前用墙钟 `performance.now()`，标签页切后台 rAF 暂停但计时照走，回来即被警戒线误判负；
- 胜利保护 `winPending`：合成出目标后的 850ms 庆祝延迟内不再被警戒线/卡槽满抢先判负；
- 卡槽相同卡片自动归组（设计文档既有规则）；警戒线判负时长对齐设计的 2 秒；
- 修复重开/下一关在同一 canvas 上重建 PixiJS 导致 WebGL 上下文丢失卡死（canvas 按局 `key` 重建）、返回主菜单双重 destroy 崩溃、`merging` 集合泄漏；
- 开发模式暴露 `window.__game`，自动化测试可 `for(...) __game.tick({deltaMS:16.67})` 快进逻辑（后台标签页 rAF 不跑，勿靠实时等待）。

## Web Game 3.0：果园远征与无尽狂欢

### 三种模式定位

1. **闯关模式**：14 个叠层关卡，负责教学、解锁水果和建立合成目标。每关必须通过至少一次物理碰撞才能获得目标水果。
2. **无尽模式**：没有关卡目标，清空卡组后立即进入下一波。波次越高，卡片数量、叠层深度和特殊牌概率越高；双黄金果王相撞会清空果箱并奖励 `100000 × 得分倍率`。
3. **Rogue 模式**：连续挑战 8 关，分数跨关继承。前 7 关胜利后从 3 件随机奇物中选择 1 件，奇物不重复出现；远征福袋不会缩短总路线数。

### 单局节奏

- **观察 3–8 秒**：辨认初始可点三消、炸弹位置和冰冻解锁方向。
- **卡槽决策 20–60 秒**：相同卡自动归组；优先完成三消，或利用炸弹连取改变顶部结构。
- **投放决策 2–4 秒**：三消水果悬停在果箱顶部，玩家可点选落点；未操作时自动对准同级水果投放。
- **物理连锁 1–6 秒**：同级水果磁吸、碰撞、合成，连续反应进入倍率窗口。
- **资源决策**：卡槽接近满时使用清顶锤；果箱拥堵时使用榨汁；同级水果被隔开时使用磁核。

### 道具体系

- **撤回**：仅撤回尚未形成三消的最后一张普通卡，保留策略纠错空间。
- **洗牌**：重排剩余卡片的位置与层级，不改变数量和水果资源。
- **榨汁**：将果箱中最高级水果降一级，主动释放空间并改变合成链。
- **清顶锤**：自动选择卡槽最需要的可见牌；同时解除冰冻或藤蔓。
- **合并磁核**：选择距离最近的一对同级水果，立即触发一次物理合成。

### 空间与视觉平衡

- 19 级水果物理半径控制在 `8–60`；大果日临时放大 15%，平时不会两三颗就堵满 396px 宽果箱。
- Roguelike 的迷你果园会在新尺寸上再缩小 14%，形成明确的容量流派优势。
- 合成箱内水果取消静态彩色圆盘与外圈，只显示水果本体和轻量投影；粒子总量限制为 190，低端手机上连锁爆发不会无限堆积。
- 三消震屏、普通合成震屏和高阶合成震屏分级，保证反馈明显但不影响投放判断。

### 防软锁与公平性

- 每个关卡开局保证 3 张同类牌处于可点区域；卡片总量均为 3 的倍数。
- 主线/远征残局使用“两两合并推演”确认是否仍可达到目标；无尽模式在卡组清空但卡槽残留时结算失败。
- 待投放水果、合并中的水果和胜利演出期间不会提前触发残局判负。
- 炸弹连取仍进入卡槽并遵守七格/八格上限，不提供无成本清牌。
- 无尽、远征和主线分别记录排行榜，避免不同计分结构互相污染。

### 有机叠层与参与感升级（2026-07-17）

- **水果 17→19 级**：新增 番茄🍅 与 香蕉🍌；16 关目标固定为 3→18 逐关 +1，每关卡池都含 `0…目标-1` 的完整阶梯，整数进位模拟保证至少合出 1 个目标；

### 甜度狂热、变异波次与温室成长（2026-07-17，第二批）
