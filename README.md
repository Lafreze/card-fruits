# 叠个果王 · Neon Orchard

一款为手机操作习惯设计的水果三消 + 物理合成网页游戏：从叠放的果卡中选出三张同类卡，生成实体水果；两个同级水果碰撞后继续升级，最终合成榴莲造型的黄金果王。

## 游戏亮点

- **PixiJS 8 游戏渲染**：游戏主画面、卡片、物理水果、星空背景与粒子效果全部运行在 WebGL/Canvas 渲染层中。
- **三路自选落果**：每次三消固定生成 1 颗同类水果，可随时选择左、中、右落点；温室“丰收培育”最多提升到 3 颗，不再由概率决定。
- **落稳再合成**：空中的同级水果不会合成，只有接触地面或已落稳果堆后才获得合成资格；点击水果仍可向同级伙伴方向弹射解开死角。
- **手游式反馈**：连击倍率、粒子爆裂、冲击波、震屏、音效、触觉反馈、警戒线与华丽结算。
- **47 关闯关模式**：整整 50 种水果从蓝莓、橄榄一路合到黄金果王；完整西瓜、香蕉、火龙果以及柿子、李子、金桔、百香果、龙眼、杨梅、人参果、蛋黄果、刺角瓜、佛手柑、仙人掌果等后期水果使用独立绘制素材。
- **甜度狂热**：三消和物理合成持续充能，满能量进入 9 秒 FEVER，得分翻倍、磁吸强化、连击窗口延长。
- **无尽模式**：完成第 2 关后解锁；卡组按波次无限补充，第 2 波起会随机改变果实尺寸、特殊牌、磁力、警戒时间、掉果数量、卡槽容量、重力或弹射强度；双果王触发彩虹清场。
- **Rogue 模式**：完成第 4 关后解锁；8 关连续挑战、36 项构筑强化，可走产量、弹射、物理庆典、牌组采摘、容量、连击或救场流派。
- **统一实用道具栏**：牌桌、果箱与增益道具全部集中在顶部道具栏；每局自带洗牌、清顶、净化、炸弹、冰冻与搅拌，温室中的 16 种补给可继续强化。
- **动态计分与清脆反馈**：卡牌层级、果实阶级、连击、模式难度和残局表现都会影响得分，并配有更轻快的果肉脆响、汁液爆裂与合成果音。
- **功能牌与复杂牌阵**：冰冻、炸弹、藤蔓、甜度、丰收、棱镜与季风牌会同时改变卡槽和果箱；大平层会拆成互相咬合的微层，同时保留经过验证的安全移除路线。
- **PostgreSQL 排行榜**：每局由服务端签发唯一记录；排行榜展示全局玩家，每位玩家只保留个人最佳成绩，避免重复记录挤满榜单。
- **响应式设计**：手机全屏游玩，桌面端保留沉浸式竖屏游戏框。

## 技术栈

- React 19 + TypeScript + Vite
- PixiJS 8 + Matter.js
- Node.js 22 + Express 5
- PostgreSQL + `pg`
- Zod、Helmet、限流与压缩中间件

## 本地运行

```bash
npm install
cp .env.example .env
npm run dev
```

游戏页面默认运行在 `http://localhost:5173`，接口服务默认运行在 `http://localhost:3000`。

没有配置 PostgreSQL 时仍可正常游玩，但排行榜会显示离线。若需要完整功能，请创建数据库并将连接地址写入 `.env` 的 `DATABASE_URL`。服务启动时会自动执行幂等表结构初始化。

## Railway 部署

1. 在 Railway 中创建项目，并连接本 GitHub 仓库。
2. 在同一项目中添加 **PostgreSQL** 服务；Railway 会向游戏服务注入 `DATABASE_URL`。
3. 游戏服务使用仓库中的 `railway.json` 与 `nixpacks.toml` 自动构建和启动。
4. 如使用 Railway 代理，设置环境变量 `TRUST_PROXY=1`。
5. 部署完成后，健康检查地址为 `/api/health`。

生产服务由 Express 同时提供静态游戏与 `/api` 接口，无需拆分前后端服务。

## 常用命令

```bash
npm run build      # TypeScript 检查与生产构建
npm test           # 服务端校验测试
npm start          # 启动生产服务
```

## 数据表

- `game_runs`：每局开始时创建，记录模式、关卡、开始与完成时间。
- `scores`：保存用户名、分数、最高连击、最大水果与游戏时长。

数据库定义位于 `database/schema.sql`。
